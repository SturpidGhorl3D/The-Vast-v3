# Система редактора портретов видов

Данный документ описывает концептуальную архитектуру модульного редактора портретов видов, позволяющего создавать уникальных существ с использованием слоистости, процедурной генерации конечностей и физики.

## 1. Концепция и дизайн-философия
Цель системы — предоставить пользователю мощный инструмент для создания внешнего вида вида (species), соблюдая баланс между простотой для новичка и глубокой настройкой для творческого игрока.

*   **Слоистость (Layering):** Портрет состоит из накладываемых друг на друга частей тела.
*   **Доступность:** Начинающий игрок использует пресеты и ползунки (авто-текстурирование). Продвинутый игрок может переопределить текстуру части тела с помощью встроенного редактора пиксельной графики.
*   **Визуальный стиль:** Основан на спрайтах в полоборота (3/4), что позволяет создавать ощущение объема при простой реализации.

## 2. Основная архитектура

### Сокеты и крепления
Каждая часть тела (например, торс) содержит набор **сокетов** — точек привязки для других частей (рук, голов, дополнительных сегментов).
*   **Тип соединения:**
    *   *Жесткое (Rigid):* Часть тела просто прикреплена.
    *   *Эластичное (Verlet/Soft):* Использует базовую симуляцию физики для имитации мягкости или эластичности соединения.
*   **Параметры сокета:** Координаты (относительно текстуры родителя), угол поворота, Z-индекс, ограничения движения (лимиты вращения).

### Физический движок частей
Каждая текстура/спрайт (или группа частей) может иметь приписанную физическую модель:
1.  **Rigid:** Твердый объект, вращающийся вокруг сокета-сустава.
2.  **Verlet:** Деформируемая структура. Используется точечная масса (mass-spring system) для визуализации мягких тканей, отростков или киберимплантов, реагирующих на движения родительского объекта.

## 3. Механики настройки

### Динамический авто-текстурёр
Для конечностей используется процедурный генератор, управляемый ползунками:
*   Длина, толщина, изгиб, количество суставов.
*   Генерация текстур на основе базового скина или паттерна (чешуя, мех, металл).

### Ручное редактирование (Canvas-based)
Если процедурный результат не устраивает, игрок может нажать кнопку "Редактировать текстуру", открывающую слой в ограниченном редакторе пиксель-арта, где он может дорисовать детали или изменить форму конечности поверх созданной авто-генератором.

### Z-уровень и порядок отрисовки
Система учитывает Z-индекс для каждой прикрепляемой части. Это необходимо для того, чтобы конечности могли находиться как *перед*, так и *за* торсом.

## 4. Технические аспекты
*   **Представление данных:** Портрет сохраняется как дерево объектов JSON, описывающее иерархию креплений, типы сокетов, ссылки на атласы спрайтов (или модифицированные текстуры) и параметры физики для каждого сегмента.
*   **Рабочая среда:** Редактор работает с спрайтовым атласом, где части тела нарезаны и подготовлены к динамическому рендерингу (через WebGL/Canvas2D).
*   **Производительность:** Так как портрет — это набор динамических спрайтов с физикой, необходимо использовать кэширование отрисовки сегментов, если они не находятся в активной анимации.
